# 최종 보완 명세 v2

## 1. 문서 목적

이 문서는 기존 `requirements.md`, `system-architecture.md`, `api-spec.md`를 실제 구현 기준으로 보완한 최종 사양서이다.

목표는 다음과 같다.

- 구현자가 중간에 다시 중요한 결정을 하지 않도록 기준을 고정한다.
- 웹 페이지 프레임워크, 백엔드 API, 데이터 모델, 백그라운드 작업, 익명 세션 정책을 하나의 문서로 통합한다.
- MVP 범위에서 실제 뉴스 수집과 AI 분석까지 동작하도록 계약을 닫는다.

이 문서가 기존 3개 문서보다 우선한다. 기존 문서와 충돌하는 항목은 이 문서를 기준으로 구현한다.

## 3. 확정 기술 스택

- 백엔드: FastAPI
- 서버 렌더링: Jinja2
- 프론트엔드: HTML, CSS, Vanilla JavaScript
- 지도: Leaflet
- 데이터베이스: MongoDB Atlas
- DB 접근: PyMongo Async API
- 스케줄러: APScheduler `AsyncIOScheduler`
- AI 분석 공급자: OpenAI API

### 3.1 시간 및 저장 기준

- 스케줄러 타임존은 `Asia/Seoul`로 고정한다.
- DB에 저장하는 모든 시각 필드는 UTC ISO 8601 문자열 또는 UTC `datetime`으로 저장한다.
- API 응답의 날짜/시간은 UTC ISO 8601 형식으로 반환한다.

### 3.2 권장 디렉토리 구조

```text
app/
  main.py
  core/
    config.py
    database.py
    session.py
    scheduler.py
  routers/
    pages.py
    news_api.py
    saved_api.py
  services/
    news_ingestion.py
    news_normalizer.py
    ai_analysis.py
    importance.py
    geo_mapping.py
  repositories/
    news_repository.py
    saved_repository.py
  templates/
    home.html
    category.html

    css/
```

위 구조는 권장안이며, 동일한 책임 분리가 유지되면 세부 경로명은 다소 달라도 된다.

## 4. 화면 및 라우트 명세

### 4.1 페이지 라우트

| 경로 | 페이지 | 설명 |
| --- | --- | --- |
| `/` | 홈 | 전체 이슈 지도, 검색, 주요 뉴스 카드 |
| `/war` | 전쟁 | 전쟁 카테고리 지도와 상세 패널 |
| `/economy` | 경제 | 경제 카테고리 지도와 상세 패널 |
| `/disaster` | 자연재해 | 자연재해 카테고리 지도와 상세 패널 |
| `/politics` | 정치 | 정치 카테고리 지도와 상세 패널 |
| `/my-articles` | 나만의 기사 | 현재 세션 저장 기사 목록 및 상세 보기 |

### 4.2 페이지 렌더링 방식

- 모든 페이지는 서버 렌더링으로 최초 진입 화면을 반환한다.
- 이후 지도 상호작용, 검색, 저장/삭제는 API를 호출하는 방식으로 동작한다.
- 페이지 최초 로드시 필요한 초기 데이터는 서버에서 주입한다.

### 4.3 페이지별 동작

#### 홈

- 지도에 모든 카테고리 핀을 표시한다.
- 상단 검색 폼은 자동 검색이 아니라 명시적 제출 방식으로 동작한다.
- 검색 결과는 지도 핀과 결과 리스트를 동시에 갱신한다.
- 주요 뉴스 카드는 중요도 내림차순, 동률이면 최신순으로 5건 노출한다.

#### 카테고리 페이지

- 해당 카테고리 기사만 지도에 표시한다.
- 핀 클릭 시 우측 상세 패널 또는 하단 상세 패널을 연다.
- 상세 패널은 기사 기본 정보와 AI 분석 상태/결과를 보여준다.
- 저장 버튼 클릭 시 저장 API를 호출하고 성공하면 버튼 텍스트를 `저장됨`으로 변경한다.

#### 나만의 기사

- 현재 세션에 저장된 기사만 노출한다.
- 목록에서 기사 클릭 시 기사 상세 및 AI 분석을 조회해 패널에 표시한다.
- 삭제 버튼 클릭 시 삭제 API 호출 후 해당 항목을 목록에서 제거한다.

## 5. 사용자 식별 및 세션 정책

### 5.1 식별 방식

- 로그인은 도입하지 않는다.
- 사용자는 익명 세션 쿠키 `gid_session`으로 식별한다.

### 5.2 쿠키 규칙

- 쿠키명: `gid_session`
- 값 형식: UUID v4 문자열
- 속성:
  - `HttpOnly=true`
  - `SameSite=Lax`
  - `Path=/`
  - `Max-Age=2592000` (30일)
  - 운영 환경에서 `Secure=true`
  - 로컬 개발 환경에서만 `Secure=false`

### 5.3 발급 및 갱신 규칙

- HTML 페이지 요청 또는 API 요청에 `gid_session` 쿠키가 없으면 서버가 즉시 신규 세션을 발급한다.
- 신규 세션이 발급된 요청은 해당 세션 ID를 그 요청 처리에도 즉시 사용한다.
- 쿠키 값이 UUID 형식이 아니면 기존 값을 무효 처리하고 새 세션을 발급한다.
- 모든 성공 응답에서 `gid_session`의 만료 시간을 30일로 연장하는 슬라이딩 만료 정책을 사용한다.

### 5.4 저장 기사 접근 규칙

- 저장 기사 조회는 현재 요청의 `gid_session`과 일치하는 데이터만 반환한다.
- 저장 기사 삭제는 현재 요청의 `gid_session`과 일치하는 데이터만 허용한다.
- 다른 세션의 저장 기사 ID를 전달하면 `404 SAVED_ARTICLE_NOT_FOUND`로 응답한다.

## 6. 기사 ID 및 식별자 정책

### 6.1 공통 원칙

- MongoDB `_id`는 내부 저장용 식별자로만 사용한다.
- 외부 API와 프론트엔드에는 `_id`를 노출하지 않는다.

### 6.2 기사 식별자

- `news.id`는 외부 노출용 기사 식별자이다.
- 형식은 `news_<uuid4hex>`로 고정한다.
- `article_id`는 항상 `news.id`를 참조한다.
- 기사 상세 API의 경로 파라미터 `{article_id}`는 `news.id`를 의미한다.

### 6.3 저장 기사 식별자

- `saved_articles.id`는 외부 노출용 저장 기사 식별자이다.
- 형식은 `saved_<uuid4hex>`로 고정한다.
- 저장 기사 삭제 API의 `{saved_id}`는 `saved_articles.id`를 의미한다.

### 6.4 외부 기사 식별자

- 외부 뉴스 공급자 식별자는 `external_id` 필드에 저장한다.
- 원문 주소는 `original_url`에 저장한다.
- 중복 판별은 `original_url` 우선, 없으면 `source + title + published_at` 보조 규칙을 사용한다.

## 7. 뉴스 수집 및 정규화 명세

### 7.1 외부 뉴스 공급자

- 1차 구현의 외부 뉴스 공급자는 `NewsAPI.org`로 고정한다.
- 사용 목적은 최신 국제 이슈 기사 수집이다.

### 7.2 수집 주기

- APScheduler가 10분마다 뉴스 수집 작업을 수행한다.
- 앱 시작 시 1회 즉시 수집을 시도한다.

### 7.3 카테고리별 수집 쿼리

수집 작업은 아래 4개 카테고리별 쿼리를 각각 수행한다.

- `war`: `war OR military OR missile OR conflict OR invasion OR ceasefire`
- `economy`: `economy OR inflation OR interest rate OR trade OR market OR tariff`
- `disaster`: `earthquake OR flood OR wildfire OR hurricane OR typhoon OR drought`
- `politics`: `election OR government OR parliament OR diplomacy OR summit OR sanctions`

### 7.4 수집 범위

- 각 카테고리별로 최근 7일 기사만 조회한다.
- 1회 수집 시 카테고리당 최대 25건까지만 가져온다.
- 중복 제거 후 `news` 컬렉션에 upsert한다.

### 7.5 정규화 규칙

외부 공급자 응답은 아래 내부 표준 스키마로 변환한다.

- `id`
- `external_id`
- `original_url`
- `title`
- `source`
- `published_at`
- `summary`
- `content`
- `country`
- `continent`
- `region`
- `category`
- `keywords`
- `lat`
- `lng`
- `importance`
- `pin_size`
- `pin_color`

### 7.6 카테고리 최종 결정 규칙

- 수집 작업에서 사용한 카테고리 쿼리 결과를 1차 카테고리로 둔다.
- 제목과 요약을 소문자 기준으로 다시 검사해 카테고리 키워드 점수를 계산한다.
- 다른 카테고리 점수가 더 높으면 그 카테고리로 재분류한다.
- 점수 동률이면 우선순위는 `disaster > war > politics > economy` 순으로 결정한다.
- 어떤 카테고리 점수도 0이면 해당 기사는 저장하지 않는다.

### 7.7 위치 보강 규칙

- 외부 응답에 좌표가 있으면 그 값을 그대로 사용한다.
- 좌표가 없고 `country`가 있으면 `data/country_centroids.json`에서 국가 중심 좌표를 매핑한다.
- `country`가 없고 `region`만 있으면 `data/region_centroids.json`에서 지역 중심 좌표를 매핑한다.
- 좌표를 끝내 결정하지 못한 기사는 DB에는 저장하되 지도 핀에서는 제외한다.

### 7.8 대륙 및 지역 보강 규칙

- `country`가 있으면 국가-대륙 매핑 사전을 사용해 `continent`를 채운다.
- `region`은 기사 제목/요약/국가 기준의 룰셋으로 결정한다.
- 허용 대륙 값은 `Asia`, `Europe`, `Africa`, `NorthAmerica`, `SouthAmerica`, `Oceania` 6개로 고정한다.

### 7.9 보존 정책

- `news` 컬렉션은 최근 30일 기사만 유지한다.
- 매일 새벽 3시 `Asia/Seoul` 기준으로 30일 초과 기사를 정리한다.

## 8. 중요도 및 핀 규칙

### 8.1 중요도 계산 규칙

중요도는 1~5 정수로 계산한다.

- 기본 점수는 2점으로 시작한다.
- 발행 후 12시간 이내면 +1
- 발행 후 72시간 이내면 추가 +1
- 제목 또는 요약에 카테고리별 고강도 키워드가 1개 이상 있으면 +1
- 제목 또는 요약에 고강도 키워드가 2개 이상 있으면 추가 +1
- 최대값은 5, 최소값은 1로 clamp한다.

### 8.2 고강도 키워드 예시

- `war`: invasion, missile, strike, nuclear, siege
- `economy`: recession, crash, rate hike, sanctions, default
- `disaster`: earthquake, tsunami, wildfire, flood, eruption
- `politics`: coup, impeachment, election crisis, martial law, sanctions

### 8.3 핀 크기 규칙

| importance | pin_size |
| --- | --- |
| 1~2 | `small` |
| 3~4 | `medium` |
| 5 | `large` |

### 8.4 핀 색상 규칙

| category | pin_color |
| --- | --- |
| `war` | `#EF4444` |
| `economy` | `#16A34A` |
| `politics` | `#EAB308` |
| `disaster` | `#F97316` |

선택된 핀은 색상을 바꾸지 않고 외곽선 또는 그림자 강조만 추가한다.

## 9. AI 분석 생성 정책

### 9.1 생성 방식

- AI 분석은 기사 상세 조회 시 즉석 생성하지 않는다.
- 뉴스 수집 후 `ai_status=pending` 상태로 저장한다.
- 별도 스케줄러 작업이 15분마다 `pending` 및 재시도 대상 기사를 처리한다.

### 9.2 AI 공급자 및 입력

- AI 공급자는 OpenAI API로 고정한다.
- 입력 데이터는 아래 필드를 사용한다.
  - `title`
  - `summary`
  - `category`
  - `country`
  - `continent`
  - `region`
  - `source`
  - `published_at`

### 9.3 AI 출력 형식

모델 출력은 반드시 JSON 객체 형식이어야 하며 다음 구조를 따른다.

```json
{
  "interpretation": "기사 의미 요약",
  "prediction": "향후 예상 동향",
  "impact": {
    "key1": "영향 설명",
    "key2": "영향 설명"
  }
}
```

### 9.4 AI 상태 필드

- `ai_status`: `pending | completed | failed`
- `ai_interpretation`
- `ai_prediction`
- `ai_impact`
- `ai_generated_at`
- `ai_attempts`

### 9.5 재시도 규칙

- 1회 작업에서 기사 최대 50건까지 처리한다.
- 실패한 기사는 1시간 후 재시도한다.
- 최대 재시도 횟수는 3회이다.
- 3회 실패 후에도 분석이 생성되지 않으면 `ai_status=failed`로 유지한다.

### 9.6 재생성 규칙

- 이미 `completed`인 분석은 24시간 내 재생성하지 않는다.
- 기사 본문 또는 요약이 갱신되면 `ai_status`를 다시 `pending`으로 바꾸고 분석을 갱신한다.

### 9.7 프론트 노출 규칙

- `pending`: "AI 분석 준비 중" 메시지 표시
- `completed`: 해석, 예상 동향, 영향 표시
- `failed`: "AI 분석을 불러오지 못했습니다" 메시지 표시

`pending`과 `failed`는 정상 상태로 간주하며, 분석 조회 API는 200 응답을 유지한다.

| `summary` | string | 기사 요약 |
| `saved_at` | datetime | 저장 시각 UTC |

### 10.3 권장 인덱스

#### `news`

- unique: `id`
- unique sparse 가능: `original_url`
- index: `category`
- index: `continent`
- index: `published_at` desc
- index: `importance` desc
- index: `ai_status`
- index: `updated_at`
- text index: `title`, `summary`
- multikey index: `keywords`

#### `saved_articles`

- unique compound: `session_id + article_id`
- unique: `id`
- index: `session_id + saved_at`
- index: `session_id + category`

## 11. 백그라운드 작업 명세

### 11.1 스케줄러 구성

- FastAPI 앱 시작 시 `AsyncIOScheduler(timezone="Asia/Seoul")`를 초기화한다.
- 앱 종료 시 스케줄러를 정상 종료한다.

### 11.2 작업 목록

| 작업명 | 주기 | 설명 |
| --- | --- | --- |
| `collect_news_job` | 10분마다 | 외부 뉴스 수집, 정규화, upsert |
| `generate_ai_analysis_job` | 15분마다 | AI 분석 생성 및 재시도 |
| `cleanup_old_news_job` | 매일 03:00 | 30일 초과 기사 삭제 |

### 11.3 시작 시 동작

- 앱 시작 직후 인덱스를 보장한다.
- 인덱스 생성 후 `collect_news_job`을 1회 즉시 실행한다.
- 첫 수집이 실패해도 서버는 기동한다.
- 다만 DB에 기사 데이터가 전혀 없으면 뉴스 조회 API는 `503 NEWS_SOURCE_UNAVAILABLE`를 반환한다.

## 13. 뉴스 API 상세 명세

### 13.1 GET `/api/news/home`

홈 화면용 전체 지도 핀과 주요 뉴스 카드를 반환한다.

#### Query Parameters

| 이름 | 타입 | 필수 | 기본값 | 최대값 | 설명 |
| --- | --- | --- | --- | --- | --- |
| `continent` | string | N | 없음 | - | 대륙 필터 |
| `keyword` | string | N | 없음 | 100자 | 제목/요약/키워드 검색 |
| `limit` | int | N | 5 | 20 | 주요 뉴스 카드 개수 |

#### 반환 규칙

- `map_pins`에는 `lat`, `lng`가 모두 있는 기사만 포함한다.
- `top_headlines`는 중요도 내림차순, 동률이면 `published_at` 내림차순으로 정렬한다.

#### Response `data`

- `map_pins`: 지도 표시용 기사 배열
- `top_headlines`: 주요 뉴스 카드 배열

### 13.2 GET `/api/news/category/{category}`

카테고리 페이지용 기사 목록을 반환한다.

#### Query Parameters

| 이름 | 타입 | 필수 | 기본값 | 최대값 | 설명 |
| --- | --- | --- | --- | --- | --- |
| `continent` | string | N | 없음 | - | 대륙 필터 |
| `keyword` | string | N | 없음 | 100자 | 검색어 |
| `sort` | string | N | `importance` | - | `importance` 또는 `latest` |
| `limit` | int | N | 50 | 100 | 최대 반환 개수 |

#### 반환 규칙

- `articles`는 카테고리 일치 기사만 포함한다.
- `sort=importance`이면 `importance desc, published_at desc` 순으로 정렬한다.
- `sort=latest`이면 `published_at desc, importance desc` 순으로 정렬한다.

### 13.3 GET `/api/news/continent/{continent}`

대륙 기준 기사 목록과 지도 표시 데이터를 반환한다.

#### Query Parameters

| 이름 | 타입 | 필수 | 기본값 | 최대값 | 설명 |
| --- | --- | --- | --- | --- | --- |
| `category` | string | N | 없음 | - | 카테고리 필터 |
| `keyword` | string | N | 없음 | 100자 | 검색어 |
| `limit` | int | N | 50 | 100 | 최대 반환 개수 |

#### 반환 규칙

- `articles`는 해당 대륙 기사만 포함한다.
- 정렬은 `importance desc, published_at desc`로 고정한다.

### 13.4 GET `/api/news/search`

통합 뉴스 검색 API이다.

#### Query Parameters

| 이름 | 타입 | 필수 | 기본값 | 최대값 | 설명 |
| --- | --- | --- | --- | --- | --- |
| `q` | string | Y | 없음 | 100자 | 검색어 |
| `continent` | string | N | 없음 | - | 대륙 필터 |
| `category` | string | N | 없음 | - | 카테고리 필터 |
| `limit` | int | N | 10 | 50 | 페이지당 개수 |
| `page` | int | N | 1 | - | 1부터 시작 |

#### 검색 기준

- `title`
- `summary`
- `keywords`
- `country`
- `region`

#### 반환 규칙

- 정렬은 텍스트 검색 점수 우선, 동률이면 `importance desc`, 그다음 `published_at desc`를 사용한다.
- 반환값에는 `query`, `total`, `page`, `page_size`, `articles`를 포함한다.
- 결과가 없으면 `success=true`, `total=0`, `articles=[]`로 반환한다.

### 13.5 GET `/api/news/{article_id}`

단일 기사 상세 정보와 상세 패널용 데이터를 반환한다.

#### 반환 규칙

- `content`가 없으면 `null`로 반환한다.
- 존재하지 않으면 `404 NEWS_NOT_FOUND`를 반환한다.

### 13.6 GET `/api/news/{article_id}/analysis`

기사 AI 분석 결과를 반환한다.

#### 반환 규칙

- 항상 `article_id`, `ai_status`, `interpretation`, `prediction`, `impact`를 포함한다.
- `pending`이면 `interpretation`, `prediction`, `impact`는 `null`이다.
- `failed`이면 `interpretation`, `prediction`, `impact`는 `null`이다.
- 존재하지 않으면 `404 NEWS_NOT_FOUND`를 반환한다.

#### Response `data`

```json
{
  "article_id": "news_xxx",
  "ai_status": "pending",
  "interpretation": null,
  "prediction": null,
  "impact": null
}
```

## 14. 저장 기사 API 상세 명세

### 14.1 POST `/api/articles/save`

현재 세션 사용자가 기사를 저장한다.

#### Request Body

```json
{
  "article_id": "news_xxx"
}
```

#### 동작 규칙

- `article_id`는 반드시 존재하는 `news.id`여야 한다.
- 동일 세션에서 같은 `article_id`를 다시 저장하면 새 문서를 만들지 않는다.
- 중복 저장 시 기존 저장 문서를 반환하고 `already_saved=true`로 응답한다.

#### Response `data`

```json
{
  "saved_id": "saved_xxx",
  "article_id": "news_xxx",
  "already_saved": false
}
```

### 14.2 GET `/api/articles/saved`

현재 세션의 저장 기사 목록을 반환한다.

#### Query Parameters

| 이름 | 타입 | 필수 | 기본값 | 설명 |
| --- | --- | --- | --- | --- |
| `category` | string | N | 없음 | 카테고리 필터 |
| `continent` | string | N | 없음 | 대륙 필터 |
| `sort` | string | N | `latest` | `latest` 또는 `category` |

#### 반환 규칙

- 현재 세션 기준 데이터만 반환한다.
- `sort=latest`이면 `saved_at desc`
- `sort=category`이면 `category asc, saved_at desc`

### 14.3 DELETE `/api/articles/saved/{saved_id}`

현재 세션의 저장 기사를 삭제한다.

#### 동작 규칙

- 현재 세션과 일치하는 `saved_id`만 삭제한다.
- 존재하지 않거나 세션이 다르면 `404 SAVED_ARTICLE_NOT_FOUND`를 반환한다.

#### Response `data`

```json
{
  "deleted_id": "saved_xxx"
}
```

## 17. 테스트 및 검수 기준

### 17.1 세션

- 첫 방문 HTML 요청에서 `gid_session` 쿠키가 발급된다.
- 첫 API 요청이 먼저 와도 `gid_session`이 발급되고 그 세션으로 저장/조회가 가능하다.
- 다른 브라우저 또는 시크릿 세션과 저장 목록이 분리된다.

### 17.2 뉴스 수집

- 스케줄러가 10분마다 정상 실행된다.
- 동일 기사가 중복 삽입되지 않는다.
- 좌표가 없는 기사는 리스트에는 보이고 지도에는 나타나지 않는다.
- 첫 수집 실패 및 캐시 없음 상태에서 조회 API는 `503 NEWS_SOURCE_UNAVAILABLE`를 반환한다.

### 17.3 검색 및 목록

- 홈 주요 뉴스는 최대 5건 기본 노출된다.
- 카테고리 페이지는 해당 카테고리 기사만 노출한다.
- 검색 API는 제목/요약/키워드/국가/지역 검색이 모두 동작한다.
- 페이지네이션에서 `page=1`이 기본이고 빈 결과도 정상 응답한다.

### 17.4 AI

- 새 기사 저장 후 `ai_status=pending`으로 시작한다.
- 스케줄러가 분석을 완료하면 `completed`로 전환된다.
- 3회 실패 후 `failed`로 유지된다.
- `pending`과 `failed` 상태에서도 상세 페이지는 깨지지 않는다.

### 17.5 저장 기사

- 같은 세션에서 같은 기사를 두 번 저장해도 문서는 1개만 존재한다.
- 저장 기사 삭제는 같은 세션에서만 가능하다.
- 나만의 기사 페이지에서 삭제 후 즉시 목록에서 제거된다.

## 19. 최종 결정 요약

- 사용자 식별은 익명 세션 쿠키 `gid_session`으로 처리한다.
- MongoDB `_id`는 내부 전용이고, 외부에는 `news.id`, `saved_articles.id`만 노출한다.
- 실뉴스 공급자는 `NewsAPI.org`, 지도는 `Leaflet`, AI 분석은 `OpenAI API`로 고정한다.
- 백그라운드 작업은 FastAPI 내부 `APScheduler`로 운영한다.
- 저장 기사 삭제 API는 `DELETE /api/articles/saved/{saved_id}`로 통일한다.
- AI `pending`과 `failed`는 에러가 아니라 정상 상태로 응답한다.

이 문서 기준으로 구현하면, 기존 문서에서 비어 있던 사용자 식별, 백그라운드 실행 방식, ID 계약, 에러 처리, AI 상태 처리까지 모두 고정된 상태로 웹 프레임워크를 완성할 수 있다.
